During my Systems Engineering Internship at CAS Severn, a local government client requested a program to automate record-keeping for their Nutanix cluster and generate structured Excel reports for internal use. As the only programmer on the team, I architected and development the software from the ground up.

Problem

The client requested a program to automate record-keeping for their Nutanix cluster by generating structured Excel reports. They were operating on a legacy version of Nutanix with an API that lacked publicly available documentation. I did not have direct access to the client’s system, which was uniquely configured, and all communication was conducted through a manager.

Solution

I reverse-engineered the legacy API from modern documentation to build a Windows-based GUI application that securely interfaced with the client's cluster and automated report generation. The application standardized cluster reporting and eliminated manual record compilation.

Technologies Used

  • C#
  • .NET Framework
  • Windows Forms (event-driven desktop application architecture)
  • Nutanix Prism API (v1 - Rest integration)
  • Newtonsoft.Json.Linq (dynamic JSON parsing)
  • OpenXML (Excel report generation)

Technical Highlights

  • Designed and developed a Windows-based GUI application, expanding the original CLI requirement to improve usability.
  • Implemented robust error handling and logging systems to enable remote troubleshooting without direct access to the client's cluster.
  • Translated client requirements into technical specifications while coordinating communication through management.
  • Independently architected and delivered production software adopted by a local government client.

Major Learnings

  • Designed and integrated with APIs, including reverse-engineering undocumented legacy systems.
  • Applied defensive programming techniques to improve reliability, validation, and error handling.
  • Architected and delivered a complete software solution independently, from requirements analysis to deployment.
  • Translated ambiguous requirements into a scalable, maintainable system design.
  • Adapted to real-world engineering constraints, including limited documentation, restricted system access, and indirect client communication.
